# Agent Key Vault 即时授权需求 V1.0

## 1. 需求目标

系统集中、安全地存储 Agent 使用的凭证。

Agent 执行任务时，只能申请一次性凭证使用权，并在人工批准后完成指定操作。

Agent 不得：

* 读取凭证明文；
* 导出凭证明文；
* 记录凭证明文；
* 长期持有凭证明文。

## 2. 核心规则

1. 首期支持以下凭证类型：

   * API Key；
   * Access Token；
   * 账号密码；
   * 证书及私钥引用。

2. 只有身份可验证的已注册 Agent 可以申请授权。

3. 每次申请由凭证所有者或指定审批人进行人工审批。

4. 一次审批仅覆盖：

   * 一个 Agent；
   * 一个任务；
   * 一项凭证；
   * 一次指定操作。

5. 授权最长有效期为 **10 分钟**，操作完成后立即失效。

6. 出现以下情况时，Agent 必须重新申请授权：

   * 重试操作；
   * 追加操作；
   * 更换目标系统。

7. Agent 不得：

   * 复制授权；
   * 转移授权；
   * 并发共享授权；
   * 跨任务复用授权。

8. 审批人可在授权归还前随时主动撤销授权。

9. 出现以下情况时，系统默认拒绝访问：

   * 未审批；
   * 审批拒绝；
   * 审批超时；
   * 上下文不匹配。

## 3. 凭证使用与回收

### 3.1 凭证使用

1. Agent 通过受控工具或代理使用凭证，不得接触凭证明文。

2. 目标系统支持临时 Token 时，优先签发一次性临时凭证。

3. 目标系统只支持固定凭证时，由受控代理代为注入和使用。

### 3.2 凭证回收

1. 静态源凭证在使用后继续保存在 Key Vault 中，系统只回收本次使用权。

2. 临时 Token、会话及派生凭证在使用后立即撤销并销毁。

3. 出现以下任一情况时，都必须执行回收：

   * 任务成功；
   * 任务失败；
   * 任务取消；
   * 任务超时；
   * Agent 异常退出。

4. 回收失败时，系统应立即：

   * 阻断后续使用；
   * 触发告警；
   * 转入人工处置。

## 4. 审批与审计

### 4.1 审批界面

审批界面必须展示以下信息：

* Agent 身份及任务标识；
* 申请原因和目标系统；
* 凭证别名，不显示凭证明文；
* 拟执行操作；
* 授权有效期；
* 风险提示。

### 4.2 审计记录

系统应记录以下事件：

* 申请；
* 批准或拒绝；
* 使用结果；
* 主动撤销；
* 自动回收；
* 销毁；
* 异常处置。

审计记录默认保留 **180 天**。

审计记录不得包含：

* 凭证明文；
* 完整 Token；
* 密码；
* 私钥材料。

## 5. 验收标准

* [ ] 未经人工批准，Agent 无法使用受保护凭证。
* [ ] Agent 获批后只能执行一次指定操作。
* [ ] Prompt、上下文、工具参数、日志和错误信息均不暴露凭证明文。
* [ ] 操作完成后，原授权再次使用必须失败。
* [ ] 授权复制、并发使用、跨 Agent 使用及跨任务使用均被拒绝。
* [ ] 任务失败、取消、超时或 Agent 退出后，授权能够自动回收。
* [ ] 审批人主动撤销后，Agent 无法继续使用。
* [ ] 静态源凭证不会被误删。
* [ ] 临时派生凭证能够正确销毁。
* [ ] 审计记录能够完整关联申请、审批、使用和回收过程。

## 6. 不在本期范围

本期不包含以下能力：

* Agent 自动审批或自我授权；
* 长期授权、批量授权或跨任务授权；
* 向 Agent 展示或导出凭证明文；
* 凭证自动轮换；
* 密码修改；
* 账号开通；
* Agent 将凭证写入以下位置：

  * 文件；
  * 环境变量；
  * Prompt；
  * 日志；
  * 长期记忆。
